
## Document template

Each REQ document must include:

- **ID** — e.g. REQ-001
- **Title** — short feature name
- **Status** — Draft | Approved | Implemented | Done
- **Priority** — Must / Should / Could
- **Effort** — S / M / L / XL
- **Description** — what and why
- **User Story** — as a [user], I want [goal] so that [reason]
- **Acceptance criteria** — testable, unambiguous conditions
- **E2E test requirements** — what scenarios must be covered by XCUITest
